1.Spring Framework
＜特徴＞
・拡張性が高い
　…フレームワークの基本的な機能の大部分が「インターフェイス」として提供されている為、必要な機能だけを外部から追加するだけなので無駄がない。
　　オブジェクト指向の基本原則を徹底してJavaを最大限に生かすことが出来るように設計されている。

・保守性が高い
　…「AOP（Aspect Oriented Proguraming）」というクラスに本質的な処理を書き、共通な処理を別に書く事が出来るプログラミングを用いることで、コードを簡素に書ける為不具合があった時に原因を特定しやすい。
　　「DI（Dependency Injection）」と呼ばれる仕組みがあり、依存性のあるプログラムは外部から取り入れるようになっていて1つ1つのプログラムを独立させやすい。

・再利用性が高い
　…「AOP」の仕組みにより共通のプログラムがまとめやすくなるので再利用が容易。

2.Play Flamework
＜特徴＞
・テスト環境が充実している。
　…環境別に画面表示を変更できるため、詳細なエラー情報を出力できる。

・軽量で動作が軽い
　…メモリ使用量が少ないのでWebアプリケーションにも対応できる。

3.JSF(Java Server Faces)
＜特徴＞
・MVC(Model-View-Controller)が採用されている
　…ModelとView、Controllerそれぞれで機能を分けることが可能なので開発効率がアップする。

・GUIがフレームワークの組み合わさっている
　…ログイン画面の制作などが容易になる。
